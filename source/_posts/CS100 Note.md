## C 语言常识

### IDE

IDE，Integrated Development Environment，中文译为集成开发环境。一般包括代码编辑器、编译器、调试器、图形用户界面等工具。

VS（Visual Studio）属于 IDE，而 VS Code 则属于代码编辑器，不内置其他功能。

### 编译器

GCC (GNU Compiler Collection) 和 Clang 是当今 C/C++ 开发领域最核心、最主流的两大编译器。

<!-- more -->

#### GCC 与 Clang 的共性

- 核心功能相同：两者都是工业级、生产可用的编译器，主要用于将 C、C++、Objective-C 等高级语言的源代码编译成可执行文件（ `EXE` 文件）。
- 极高的参数兼容性：为了方便开发者迁移，Clang 在设计之初就刻意高度兼容了 GCC 的命令行参数。绝大多数 GCC 的常用参数在 Clang 中完全适用且行为一致。
- 支持最新语言标准：两者都非常积极地跟进和实现 C 和 C++ 的最新标准。
- 跨平台与开源：两者都是开源项目，可以在 Windows、Linux、macOS 等多种操作系统和 x86、ARM、MIPS 等多种硬件架构上运行。

#### GCC 与 Clang 的区别

- 架构设计与模块化程度
  - GCC：高度耦合的单体架构，很难将其中的某一部分单独剥离出来用于其他工具。
  - Clang：基于 LLVM (Low Level Virtual Machine) 架构。严格来说，Clang 只是一个前端（Frontend），负责词法分析、语法分析并生成中间代码（LLVM IR）；而 LLVM 负责后端的优化和机器码生成。

### 执行机制

`C` 文件本质上只是一个文本文档，必须通过编译器转化为 `EXE` 文件才能运行。

### main函数签名规范

#### 返回值规范

- 必须返回 `int`
- 正常执行并成功退出时应该返回0.

#### 参数列表规范

##### 无参数写法

```c
int main(void)
```

##### 带参数写法（接受命令行输入）

```c
int main(int argc, char *argv[])
```

`argc` (Argument Count)：代表命令行参数的总个数。

`argv` (Argument Vector)：是一个字符指针数组，保存了实际的参数字符串。其中 `argv[0]` 通常是程序的运行名称或路径，`argv[1]` 开始是用户实际传入的第一个参数。

## C 语言基本数据类型

### 常见数据类型及其典型大小

- `char`: 8 bits
- `short`: 16 bits
- `int`: 32 bits
- `long`: 32/64 bits
- `long long`: 64 bits
- `float`: 32 bits
- `double`: 64 bits

1 byte = 8 bits

### unsigned

单独的 `unsigned` 是 `unsigned int` 的简写。

除了 `char` 以外，其他表示整数的数据类型都默认为 `signed`。

`char` 的符号性是由定义实现的。在不同的编译器与硬件平台上，char的默认符号性既有 signed 又有 unsigned。

### 类型大小的标准

- 平台依赖性：类型的实际大小并非绝对固定，而是依赖于具体的平台和编译器，你可以使用 `sizeof(type)` 来查看其具体占用的空间。
- 类型的相对宽度：C语言标准保证了类型的空间大小递增顺序，即 `1 == sizeof(char) <= sizeof(short) <= sizeof(int) <= sizeof(long) <= sizeof(long long)` （ `sizeof()` 的返回值是 byte）。
- 类型的绝对宽度：C语言标准也保证了类型的最低空间大小，`char` 至少 1 byte，`short` 和 `int` 至少 2 bytes，`long` 至少 4 bytes，`long long` 至少 8 bytes。
- 有符号与无符号：同一种类型的有符号（signed）和无符号（unsigned）版本占据相同大小的空间，即 `sizeof(signed T) == sizeof(unsigned T)`。

### 声明与初始化

C语言是一种静态类型语言，变量在声明之后类型无法更改。变量在使用前必须先声明。

#### 变量内存的初始状态

如果非全局变量声明后未被初始化，其内部的值是未定义（随机值）。全局变量则会自动初始化为 0。

##### 作为局部变量的数组的初始化

- 完全初始化：提供与数组长度相等的元素序列，例如 `int arr[3] = {1, 2, 3};`。
- 部分初始化：只提供前几个初始值，例如 `int arr[3] = {1, 2};`。此时前两个元素是1和2，未显式赋值的剩余元素会被编译器自动统一初始化为0。
- 全零初始化：利用部分初始化的特性，最常用的快捷数组清零方法是 `int arr[3] = {0};`，这能确保局部数组内的所有元素起始状态均为确定的0。

### 浮点数的注意事项

由于浮点数的计算精度误差难以避免，判断两个浮点数是否相等时，通常通过判断它们差的绝对值是否小于等于 $\epsilon$（例如 `fabs( double_1 - double_2 ) < 1e-8` ）来间接判断。

## 表达式运算与类型转换

### 隐式类型转换

1. **整数提升**：所有比 `int` 还要小的数据类型，只要它们参与算术运算，第一时间统统被自动提升为 `int`（`int` 装不下则提升为 `unsigned int`）。
2. **寻常算术转换阶梯**：如果经过整数提升后，两个操作数的类型依然不同，编译器就会进行向上的类型转换。转换次序从高到低依次为 `long double`, `double`, `float`, `unsigned long long`, `long long`, `unsigned long`, `long`
   , `unsigned int`, `int`。

### 无符号整数溢出

无符号数永远不能小于 0。如果我们让一个值为 0 的 `unsigned` 变量减去 1，它不会变成 -1，而是会变成 `unsigned` 类型能表示的最大值。

```c
unsigned int a = 0;
a = a - 1;
// 此时 a 的值变成了 4294967295 (如果是 32位)
```

在隐式类型转换时也会出现相似的情况。比较 `int a = -1` 和 `unsigned int b = 1` 时，`a > b` 是成立的！

## 字符类型与ASCII码

`char` 类型在底层作为整型处理，可以直接参与算术运算。

### 转义字符

- 核心控制符：`\n` 换行，`\t` 制表符，`\r` 回车，`\b` 退格，`\a` 响铃
- 标点符号转义：`\"`, `\'`, `\\`
- 内存与底层数据控制：
  - `\0`：字符串的结束标志。
  - `\ooo`：直接通过八进制数值来表示一个字符。（例：编译器会将代码中写的 `'\101'` 理解为字符 `'A'`）
  - `\xhh`：直接通过十六进制数值来表示一个字符。`x` 是固定前缀，`hh` 代表十六进制数字。（例：`'\x41'` 即代表字符 `'A'`）

## 未定义行为

实现定义行为 $\neq$ 未定义行为

实现定义行为完全合法，在同一平台与编译器下确定且一致；而未定义行为绝对非法，结果完全不可预测（可能崩溃，可能正常，可能改变时空）。

经典未定义行为：

- 有符号整数溢出：`INT_MAX + 1`。（注意：无符号整数溢出是定义良好的行为。）
- 数组越界访问：访问 `int arr[5]` 的 `arr[10]`。
- 解引用空指针或野指针：试图读取 `NULL` 或未赋值指针指向的内存。
- 除以零：如 `int a = 5 / 0;`。
- 变量未初始化就使用：读取一个没有赋初值的局部变量。

## C 语言数组基础

### 数组的类型与长度

在 C 语言中，数组的类型是由其元素类型和包含的元素个数共同决定的。如 `int a[3]={1,2,3}` 的类型是 `int [3]`，而不是 `int *`。

### 数组名的左值属性

数组名是一个不可修改的左值。这意味着

```c
int a[3] = {1, 2, 3}, b[3] = {2, 3, 4};
a = b;
```

是非法的；

```c
int a[3];
a={1,2,3};
```

也是非法的。

## 指针基础

### 指针指向什么

对于一个常规类型变量，指针中存放着这个变量的地址。

例如，对于

```c
int var = 1;
int* ptr;
ptr = &var;
```

我们就可以认为 `ptr` 中存放着 `var` 的地址。

### 理解指针——不过是一种变量

任何变量都有一个所在地址，也有一个存储内容。比如说，`var` 的所在地址就是 `&var` （不妨假定为 `0x00007ffee6b2a8fe`），而存储内容就是 1。而 `ptr` 也是一个变量，也有所在地址和存储内容。比如说，`ptr` 的所在地址是 `0x00009178d4943abc`，而存储内容是 `0x00007ffee6b2a8fe`。

调用变量名通常会直接得到变量名里存放的值。所以，调用 `ptr` 时会自动得到 `0x00007ffee6b2a8fe`，因此 `*ptr` 与 `*(0x00007ffee6b2a8fe)` 是等价的。

变量通过 `&` 获取所在地址，地址通过 `*` 找到对应的变量；而变量与存储内容之间是一个单向的箭头，调用变量会自动得到存储内容。请记住这个关系。这将对接下来的理解很有帮助。

### 星号(\*)的含义

- 在声明指针时 `*` 作为类型标识符出现。如 `int* ptr` 就意味着声明了一个类型为 `int*`（一级指针），变量名为 `ptr` 的指针。
- 在执行语句中 `*` 作为解引用操作符出现。借鉴上面的指向关系，可以认为在执行 `*ptr` 时，先发生了 `ptr` 自动得到存储内容（一个地址），然后 `*` 解引用了这个地址得到了这个地址对应的变量。
- 在解引用之前建议判断指针是否为 `NULL` 以避免未定义行为。

### 与号(&)的含义

`&` 号是取地址操作符。借鉴指向关系，可以认为 `&` 就是逆着箭头移一层，即 `&var` 表示 `var` 这个变量的所在地址。

### 比较 \* 和 &

```c
int var1 = 1, var2 = 2;
int* ptr;
```

如果执行 `ptr = &var1` 语句，那么 `ptr` 将获取 `var1` 的读写权限。例如，可以通过 `*ptr = var2` 来使 `var1` 的值改变为 `var2` 对应的 2.

如果执行 `*ptr = var1` 语句——这是一个未定义行为！因为我们并没有给 `ptr` 分配地址，所以它是一个野指针，指向一个随机的内存地址。这样做很可能导致运行错误。

那么，如果我们纯粹想给指针赋值，我们该怎么办呢？通常的办法是手动申请内存：

```c
int* ptr = malloc(sizeof(int));
if (ptr != NULL)
  *ptr = 1;
```

使用完记得用 `free(ptr);` 释放内存。

你可以理解为，手动申请内存时程序生成了一个你不知道名字的变量（`abc123def456ghi789`之类），然后指针 `ptr` 存放着了那个不知道名字的变量的地址。这有助于你套用之前的框架解决问题。

### 常见误区：使用 `ptr` $\rightarrow$ `var` $\rightarrow$ `1` 逻辑思考问题

如果使用这种逻辑，那么 `&&var` 就将是合法的。你甚至可以使用 `&&&&var` 之类的东西。

然而，`&&var` 不是合法的。我们可以举出这样的例子：

```c
int* ptr1=&var;
int* ptr2=&var;
```

那么，当我们执行 `&&var` 的时候，程序应该找到 `ptr1` 的地址还是 `ptr2` 的地址呢？

上面我们从悖论的角度表明为什么不应该采取 `ptr` $\rightarrow$ `var` $\rightarrow$ `1` 这样的逻辑，而下面我们还可以阐明为什么上文中的逻辑更好——使用上文提到的正确逻辑可以解释为何 `&&var` 天然是不合法的。从“地址-变量-值”的视角看，`&var` 得到 `var` 所在的地址，但是这个地址是一个固定的值而非变量，根本无法对其使用取地址符 `&`。

## 数组与指针

### 数组名退化

在大多数表达式和赋值中，数组名会自动退化（Decay）为指向其首元素的指针。因此可以直接将数组名赋值给指针，例如 `int *pa = a;`。

#### 函数参数申明时的数组

在函数的参数声明处，写成 `int* array`、`int array[]` 或是 `int array[6]` 没有任何区别，编译器均会将其视为一个普通的 `int*` 指针类型。

### 指针运算与数组下标等价

通过指针也可以通过下标访问内存。例如，`ptr[i]` 就与 `*(ptr + i)` 完全等价。

## 复杂指针与多维数组

### 指针数组与数组指针

C 语言的声明语法遵循了“声明模仿使用”的核心原则。例如，当声明 `int a[10]` 时，C 语言的理解逻辑是这样的：

- 如果我在代码中写出表达式 `a[i]`（这是使用它的方式）。
- 那么这个表达式 `a[i]` 求值后的最终数据类型是 `int`。

借助这个思路，我们可以看一看 `int *ptr[5]` 和 `int (*ptr)[5]` 的差别。

运算符优先级是先看 `[]`，再看 `*`。因此，`*ptr[5]` 也是先看 `ptr[5]`，再看 `*(ptr[5])`。解引用得到 `int` 类型，所以 `ptr[5]` 肯定是 `int*` 类型。因此，`ptr` 就是一个所有元素都是 `int*`类型的数组，也就是指针数组。

`(*ptr)[5]` 先看 `*ptr`，再看 `(*ptr)[5]`。取 `(*ptr)` 的第五位得到 `int`，所以 `*ptr` 是一个数组。因此，`ptr` 就是一个指向某个数组的地址的数组指针。

#### 数组指针与整型指针

```c
int arr[3]={1,2,3};
int (*ptr)[3];
ptr=&arr;
printf("%d",(*ptr)[2]);
```

和

```c
int arr[3]={1,2,3};
int *ptr;
ptr=arr;
printf("%d",ptr[2]);
```

有何区别？

事实上，这两段代码的输出是一样的。但是，它们在 C 语言的类型系统、指针的语义级别，以及指针算术运算的步长上有着本质的区别。

第一段代码：

- 指针的类型：`int (*)[3]`
- 语义与赋值：`&arr` 表示对整个数组取地址。虽然它的数值和首元素地址一样，但类型是“指向含有3个元素的数组的指针”。
- 如何访问：因为 `ptr` 指向的是“整个数组”，所以必须先解引用 `*ptr` 来获取到原数组本身，再通过 `[2]` 获取第3个元素。
- 步长：如果执行 `ptr + 1`，内存地址会跨越3个 `int` 的大小，也就是直接跳过这一整个数组。

第二段代码：

- 指针的类型：`ptr` 的类型是 `int *`。
- 语义与赋值：在赋值语句 `ptr = arr;` 中，数组名 `arr` 发生了隐式退化，变成了指向其第0个元素的指针（等价于 `&arr[0]`）。
- 如何访问：`ptr[2]` 是 C 语言中指针偏移的语法糖，它在底层会被编译器转换为 `*(ptr + 2)`，意思是从 `ptr` 当前指向的位置开始，向后移动2个 `int` 的距离，然后取出里面的值。
- 步长：如果执行 `ptr + 1`，内存地址只会跨越1个 `int` 的大小，也就是指向数组的下一个元素 `arr[1]`。

### 多维数组的函数传参退化

当二维数组（如 `int a[5][10]`）作为函数参数时，它同样遵循退化规则，但仅会忽略第一维的长度，退化为指向“一维数组的指针”（即 `int (*)[10]`）。因此，形参声明为 `int a[5][10]` 和 `int (*a)[10]` 的函数签名是完全相同的。

#### 多维数组不是多级指针！

多维数组在底层的架构是扁平的。比如，当我们声明 `int a[5][10]` 的时候，事实上内存里存储的是一个有50个 `int` 的长条。

从“声明模仿使用”的角度，`int **p` 意味着 `**p` 后期望得到一个 `int`。所以，`*p` 存储的也是一个指针。而如果我们传入的 `a[5][10]` 退化成了指向 `a[0][0]` 的指针，那么在第一次解引用中就会解到 `a[0][0]` 的真实值。然而，`a[0][0]` 中存储的是 `int` 类型而非 `int *` 类型，因此在第二次解引用的时候会发生严重的错误。所以，绝对不应该在函数传参时定义多维指针来传入多维数组。

## 函数

### 函数的定义与声明

- 函数的定义：包含了函数体，即函数内部具体执行的代码逻辑。一个函数在整个程序中只能有一个定义。
- 函数的声明：仅包含返回类型、函数名以及参数的类型，并且以分号结尾。一个函数可以被声明多次。
- 可见性原则：函数的定义本身也是一种声明。当一个函数被调用时，必须保证它的声明在当前位置之前是可见的。

### 函数参数的求值顺序

在类似 `printf("%d, %d\n", f(), g());` 的表达式中，C 语言标准没有规定同一层级参数的求值顺序。这意味着编译器可以自行决定是先执行 `f()` 还是先执行 `g()`。

虽然参数之间的求值顺序未定，但在真正进入 `printf` 函数内部执行打印之前，所有的参数（包括 `f()` 和 `g()`）都必须已经完全执行完毕。因此，在它们执行结束前，`printf` 不会输出任何内容。

## C 语言字符串

### 没有 `string`

在 C 语言中，字符串的本质是在内存中连续存储的字符数组，并且以空字符 `\0` 作为结束标志。

### 数组初始化

字符串 `hello` 包含5个可见字符，加上隐藏的 `\0`，总共占用6个字节的空间。

如果声明 `char str[5] = "hello";`，数组大小仅有5个字节，无法容纳末尾的 `\0`。此时 `str` 不算作一个合法的C字符串。如果用 `printf("%s", str)` 打印它，程序会越界读取内存直到偶然遇到一个 `\0`，这会导致未定义行为。

### 读取字符串

#### `fgets`

`fgets(str, size, stdin)` 用于从标准输入读取一行。

它最多读取 `size - 1` 个字符，从而将最后1个字节保留给结尾的 `\0` 防止缓冲区溢出。

如果在达到 `size - 1` 限制前遇到了换行符 `\n`，它会将 `\n` 也读入到字符串中，并在 `\n` 之后加上 `\0`。

#### `scanf`

`scanf("%s", str)` 会忽视所有前导空白字符，并在再次遇到空白字符时停止读取。

`scanf` 不检查目标数组的大小。如果输入的字符数量超过了数组容量，多出的字符会覆盖相邻的内存区域从而导致未定义行为。不过，可以使用 `%Ns` 来限制读取长度，例如 `scanf("%19s", str);`（假设数组大小为20，留1个位置给 `\0`）

#### `getchar`

`getchar` 可以一个一个读取字符。不过，一定要记住在以 `getchar` 方式读取的字符串后补 `\0`。

#### 输入缓冲区

输入的字符按照输入次序在缓冲区排队等待。读取函数遇到需要读入的就拿走，遇到不需要读入的就停止。比如说，`scanf` 就会把标志结束的 `\n` 留在缓存区。

### 输出字符串

#### `printf`

`printf` 可以在同一个语句中混合输出字符串、整数、浮点数等各种类型的数据。

因为 `printf` 在运行时需要解析格式化字符串（查找 `%` 符号），所以如果只是单纯输出一个字符串，它的开销比其他专用的字符串输出函数要大。

#### `puts`

`puts(str)` 不需要解析格式化字符，直接将字符串输出到屏幕，速度比 `printf` 快。在输出完字符串的可见字符后，`puts` 会自动在末尾追加一个换行符 `\n`。

#### `fputs`

向屏幕输出用 `fputs(str, stdout);`，向文件输出用 `fputs(str, file_pointer);`。

`fputs` 可以向任何文件流（包括标准输出流 `stdout`）输出。

`fputs` 不会自动换行。

#### `putchar`

`putchar` 只输出单个字符。尽管底层原理上高效，但在极少数极端性能敏感的场景下不如一次性写入一整块内存快。

### 处理字符串

#### 求字符串长度

`strlen(str)` 返回 `str` 不包含 `\0` 的实际长度。

#### 字符串复制

`strcpy(dest, src)` 将 `src` 指向的字符串复制到 `dest` 所在的内存空间。可能发生缓存区溢出。

`strncpy(est, src, n)` 只复制 `src` 的前 n 个字符。如果 `src` 的长度大于等于 n，需要手动在 `dest` 末尾添加 `\0`（`dest[sizeof(dest) - 1] = '\0';`）。

#### 字符串拼接

`strcat(dest, src)` 把连同 `\0` 的整个 `src` 拼接到 `dest` 后。
`strncat(dest, src, n)` 只拼接 `src` 的前 n 个字母到 `dest` 后。总会在末尾自动追加 `\0`。

#### 字符串比较

`strcmp(str1, str2)` 用于字符串比较，相等返回0，前者小于后者返回负数，前者大于后者返回正数。
`strncmp(str1, str2, n)` 只比较前 n 个字符，返回值规则与 `strcmp` 相同。

#### 字符串查找

`strchr(str, c)` 在字符串中查找某个字符 `c` 第一次出现的位置。返回指向该字符的指针。如果没找到，返回 `NULL`。

`strstr(haystack, needle)` 在字符串 `haystack` 中查找子字符串 `needle` 第一次出现的位置。返回指向子串起始位置的指针。如果没找到，返回 `NULL`。

#### 混合拼接字符串

`sprintf(str, format, ...)` 以 `format` 格式将后续内容组织起来存入 `str` 中，类似 `printf`。

`snprintf(str, size, format, ...)` 只写入 `size -1` 个字符，并在最后一位补上 `\0`。

## 动态内存管理

### `malloc`

声明为 `void *malloc(size_t size);`。它尝试在堆区分配连续的 `size` 字节内存。如果申请的内存过大导致系统无法满足，`malloc` 不会立刻让程序崩溃，而是会返回一个空指针 `NULL` 以提示分配失败。

`void *` 被称为泛型指针，它本身就可以被称作一种返回类型。在C语言中，`void *` 可以自动隐式转换为任何类型的指针。这个类型是在编译期确定的，而不是在运行时决定。

### `free`

语法为 `free(ptr)`

传递给 `free` 的指针必须是之前由 `malloc` 或其他分配函数返回的起始内存地址。`free(NULL)` 是安全的。

最好在 `free` 后手动将指针设为 `NULL` 以防范“悬垂指针”。
